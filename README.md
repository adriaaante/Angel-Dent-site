# Angel-Dent — улучшенный сайт

Готовая к деплою статическая версия сайта стоматологии «Ангел-Дент» в Реутове,
собранная по итогам аудита (исправление 404 в меню, опечаток, открытие
лицензии в robots, добавление JSON-LD микроразметки, отдельных посадочных
на все услуги, формы записи и др.).

Никакого билда — это чистый HTML + CSS + JS. Хостить можно где угодно:
Reg.ru, Beget, Cloudflare Pages, Vercel, Netlify, Yandex Object Storage и т.п.
Просто залейте папку целиком на хостинг.

## Что нужно сделать перед запуском

1. **Загрузить фото врачей** в `assets/img/doctors/` и заменить путь
   `placeholder.svg` на реальные имена файлов в `index.html`,
   `doctors/index.html` и индивидуальных карточках (ищите `TODO:`).
   Рекомендуется: 600×750 px, JPG/WebP, до 80 KB.

2. **Уточнить и заменить плейсхолдеры:**
   - ИНН, ОГРН и номер лицензии в подвале каждой страницы (`5012090909`,
     `1145012004000`, `ЛО-50-01-009999`) — поменять на ваши реальные.
   - Точное отчество главврача — сейчас «Эмиль Дилгамович» (в исходном
     сайте встречалось два варианта, нужно подтвердить).
   - Точные цены в прайсах на страницах услуг — заменены ориентировочными.
   - Email `info@angel-denta.ru` — если другой, поменять везде.

3. **Подключить форму к бэкенду / CRM.** Сейчас формы записи и
   обратного звонка отправляют payload в `console.log()` — найдите в
   `assets/js/main.js` строку `// fetch('/api/lead'` и подставьте свой
   webhook (Tilda Forms, Yclients, Bitrix24, AmoCRM, Make/Zapier).

4. **Метрика.** Счётчик Яндекс.Метрики уже подключён с ID `100658497`
   (взято из аудита). Если ID другой — заменить в `index.html` и
   `assets/js/main.js`.

5. **301-редиректы.** При деплое настройте редиректы со старого домена
   `angel-dent.ru` → `angel-denta.ru` и с устаревших URL Tilda
   (`/hiryrgia`, `/breket`, `/chistka` и т. д.) на новые
   (`/services/khirurgiya.html` и т. д.). На большинстве хостингов это
   делается через `.htaccess` или панель управления.

## Деплой на Tilda (если используете Tilda)

Tilda не поддерживает кастомный HTML целиком, но можно:
1. Создать пустые блоки T123 (HTML) и вставить туда содержимое внутри `<body>`.
2. Перенести стили в зональные блоки T123 в head через настройки сайта.

Проще — переехать на обычный хостинг (10× быстрее по скорости загрузки).
