# `_materials` — внутренние материалы (НЕ выкладываются на сайт)

Здесь хранятся исходные рабочие материалы клиники, чтобы они всегда были под
рукой в репозитории, но **не попадали на хостинг** (сайт angel-denta.ru на
reg.ru).

## Почему это не уходит на хостинг

Сайт деплоится скриптом `scripts/deploy.sh` — он делает `rsync` из репо в
публичную папку Apache `~/www/angel-denta.ru/`. Эта папка исключена из выгрузки
в списке `EXCLUDES` внутри `scripts/deploy.sh`:

```bash
--exclude='_materials/'
```

Так что файлы лежат в git (видны в репозитории, переживают пересоздание
окружения), но Apache их не отдаёт. Префикс `_` в имени — та же конвенция
«не для публикации», что и у `assets/img/portfolio/_originals/`.

> Если заведёшь сюда новую подпапку — выгрузка уже её не тронет (исключена вся
> `_materials/` целиком). Менять `deploy.sh` под каждую подпапку не нужно.

## Структура

```
_materials/
├── announcements/      # объявления: INDEX.md (ссылки на медиа в Drive) + STYLE.md
├── booklets/           # буклеты
└── implant-passport/   # паспорт имплантов
    ├── Паспорт_имплантов_Ангел-Дент.pdf   # оригинал
    ├── preview-page-1.png / preview-page-2.png
    └── DESIGN.md                          # спецификация макета и палитры
```

При добавлении новых типов материалов — заводи новую подпапку здесь.

## Где хранить тяжёлое медиа (картинки/видео)

Правило: **тяжёлые бинари (фото, видео) — на Google Drive, в репозитории — только
ссылки и тексты.** git хранит всю историю, поэтому большие файлы навсегда раздувают
репо и тормозят клонирование. Лёгкие исходники (PDF, превью, иконки) — можно в репо.

Пример — `announcements/INDEX.md`: таблица объявлений со ссылками на файлы в Drive.
Сам PDF паспорта оставлен в репо как исключение (он лёгкий, ~140 КБ, и это финальный
печатный исходник).
